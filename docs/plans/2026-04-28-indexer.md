# Reference Indexer (Ponder) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a Ponder-based indexer that watches Taiko Mail's events on Taiko mainnet and exposes a GraphQL API for fast inbox / outbox / key / fee / trust / delegate queries.

**Architecture:** Ponder ingests `MailSent`, `KeyPublished`, `FeeUpdated`, `Trusted`, `DelegateRegistered`, `DelegateRevoked` events. Postgres persists; GraphQL endpoint is the public API. Single Docker image, easy to self-host. Optional from the protocol's perspective — clients fall back to RPC if no indexer is configured.

**Tech Stack:** Ponder 0.7+, Postgres 16, Docker, TypeScript.

**Spec source:** `docs/superpowers/specs/2026-04-28-taiko-mail-design.md`. Depends on Plan 1 ABI artifact + deployed contract address.

### Task 2: Schema definition

**Files:**
- Modify: `indexer/ponder.schema.ts`

- [ ] **Step 1: Write schema**

```ts
import { onchainTable } from "ponder";

export const mails = onchainTable("mails", (t) => ({
  id: t.text().primaryKey(),                  // `${tx}-${logIndex}`
  txHash: t.hex().notNull(),
  blockNumber: t.bigint().notNull(),
  blockTimestamp: t.bigint().notNull(),
  sender: t.hex().notNull(),
  recipient: t.hex().notNull(),
  contentRef: t.hex().notNull(),
  valueGwei: t.integer().notNull(),
}));

export const keys = onchainTable("keys", (t) => ({
  id: t.text().primaryKey(),                  // `${owner}-${keyNonce}`
  owner: t.hex().notNull(),
  keyNonce: t.integer().notNull(),
  pub: t.hex().notNull(),
  publishedAt: t.bigint().notNull(),
}));

export const feeUpdates = onchainTable("feeUpdates", (t) => ({
  id: t.text().primaryKey(),
  recipient: t.hex().notNull(),
  valueGwei: t.integer().notNull(),
  blockTimestamp: t.bigint().notNull(),
}));

export const trustChanges = onchainTable("trustChanges", (t) => ({
  id: t.text().primaryKey(),
  recipient: t.hex().notNull(),
  sender: t.hex().notNull(),
  trusted: t.boolean().notNull(),
  blockTimestamp: t.bigint().notNull(),
}));

export const delegates = onchainTable("delegates", (t) => ({
  delegate: t.hex().primaryKey(),
  owner: t.hex().notNull(),
  clientId: t.hex().notNull(),
  registeredAt: t.bigint().notNull(),
  revokedAt: t.bigint(),
}));
```

### Task 4: Event handlers

**Files:**
- Modify: `indexer/src/index.ts`

- [ ] **Step 1: Implement all 6 handlers**

```ts
import { ponder } from "ponder:registry";
import { mails, keys, feeUpdates, trustChanges, delegates } from "ponder:schema";

ponder.on("TaikoMail:MailSent", async ({ event, context }) => {
  await context.db.insert(mails).values({
    id: `${event.transaction.hash}-${event.log.logIndex}`,
    txHash: event.transaction.hash,
    blockNumber: event.block.number,
    blockTimestamp: event.block.timestamp,
    sender: event.args.sender,
    recipient: event.args.recipient,
    contentRef: event.args.contentRef,
    valueGwei: Number(event.args.valueGwei),
  });
});

ponder.on("TaikoMail:KeyPublished", async ({ event, context }) => {
  await context.db.insert(keys).values({
    id: `${event.args.owner}-${event.args.keyNonce}`,
    owner: event.args.owner,
    keyNonce: Number(event.args.keyNonce),
    pub: event.args.pub,
    publishedAt: event.block.timestamp,
  });
});

ponder.on("TaikoMail:FeeUpdated", async ({ event, context }) => {
  await context.db.insert(feeUpdates).values({
    id: `${event.transaction.hash}-${event.log.logIndex}`,
    recipient: event.args.recipient,
    valueGwei: Number(event.args.valueGwei),
    blockTimestamp: event.block.timestamp,
  });
});

ponder.on("TaikoMail:Trusted", async ({ event, context }) => {
  await context.db.insert(trustChanges).values({
    id: `${event.transaction.hash}-${event.log.logIndex}`,
    recipient: event.args.recipient,
    sender: event.args.sender,
    trusted: event.args.trusted,
    blockTimestamp: event.block.timestamp,
  });
});

ponder.on("TaikoMail:DelegateRegistered", async ({ event, context }) => {
  await context.db
    .insert(delegates)
    .values({
      delegate: event.args.delegate,
      owner: event.args.owner,
      clientId: event.args.clientId,
      registeredAt: event.block.timestamp,
      revokedAt: null,
    })
    .onConflictDoUpdate(() => ({
      owner: event.args.owner,
      clientId: event.args.clientId,
      registeredAt: event.block.timestamp,
      revokedAt: null,
    }));
});

ponder.on("TaikoMail:DelegateRevoked", async ({ event, context }) => {
  await context.db
    .update(delegates, { delegate: event.args.delegate })
    .set({ revokedAt: event.block.timestamp });
});
```

## Self-review

- **Spec coverage:** all six event types from the contract have handlers; all five entity tables from the spec exist with the field names described.
- **Placeholders:** none. The `null` revokedAt is intentional (set when DelegateRevoked fires).
- **Type consistency:** `valueGwei` is `integer` everywhere (matches contract `uint32`). `keyNonce` is `integer`. Hex fields use `t.hex()`. Byte-checksum casing for addresses matches Ponder's lowercased default; clients querying must lowercase.
